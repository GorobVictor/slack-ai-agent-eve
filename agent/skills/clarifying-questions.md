# Clarifying Questions

When the client's request is unclear, incomplete, contradictory, or missing important context, ask follow-up questions before taking action.

## Workflow

1. Identify the smallest set of missing details that would change the answer, implementation, or next action.
2. Ask 1-3 focused questions at a time.
3. If a sensible default exists, state it briefly and ask the client to confirm it.
4. Do not invent requirements, hidden constraints, data, credentials, or business intent.
5. Continue only after the client answers or explicitly confirms the proposed default.

## Good Questions

- Ask about the decision that affects the outcome most.
- Offer concrete options when that helps the client answer faster.
- Keep the wording short and specific.

## Avoid

- Long questionnaires for details that are not needed yet.
- Proceeding with assumptions that could change behavior, data access, cost, security, or user-visible results.
- Asking for information already available in the conversation, repository, or runtime context.
